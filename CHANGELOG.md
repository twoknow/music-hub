# Music Hub 更新日志 (Changelog)

## [Unreleased] - 2026-03-06
**Windows-first 可用性收口与项目定位重写**

这一轮更新的重点不是继续堆功能，而是把项目真正收口成一个可以公开展示、可以稳定使用、也方便继续演化的 Windows-first 工具。

### 核心修复
- **修复命令面断裂**：补齐 `stats`、`undo`、`session save/load/list/delete`、`export/import` 等命令与底层接口之间的错位问题。
- **播放逻辑收口**：`m play` 恢复为优先复用主播放器；`m layer` 继续负责有意识的叠加播放；`m slots`、`m stop`、`m next` 的输出与行为更一致。
- **去掉危险的数据库回退假象**：`current`、`good`、`bad`、`next` 不再在“没有活跃播放器”时拿数据库里的历史播放冒充“当前播放”。
- **修复跨 profile 串线**：mpv IPC pipe 改为按 profile 派生，不同 `MUSICHUB_HOME` 之间不再互相切歌、串写历史或污染推荐数据。
- **修复 radio 种子策略**：`m radio` 不再被导入样例里的假 YouTube URL 卡死；相关视频抓取失败时会自动退回到 `title + artist` 搜索。

### 工程与测试
- **测试收口**：补齐 `radio`、session、快照安全性等回归测试，全量测试通过。
- **pytest 收集修复**：新增 `pytest.ini`，避免根目录测试误扫 `data/tmp` 导致权限报错。
- **备份接口兼容层补齐**：`backup.py` 暴露 CLI 需要的包装函数，方便导出与恢复数据。

### 文档与 GitHub
- **README 完全重写**：新的首页文案强调这是一个面向 Windows、面向技术小白、同时欢迎被移植到 macOS 的极简本地音乐工具。
- **仓库元信息更新**：补上 GitHub 仓库简介与 Topics，便于公开展示和检索。

## [v0.2.0] - 2026-03-05
**架构重构与流媒体化升级**

本次更新对核心播放、搜索和用户互动模块进行了深度重构，使 Music Hub 从一个本地脚本升级为了具备专业流媒体体验的个人音乐中枢。

### 🚀 性能与底层优化
- **光速搜索**：在 `yt-dlp` 调用中引入 `--flat-playlist` 参数，跳过繁重的 JavaScript 解析，使搜索速度从 10秒+ 缩减至 1-2 秒。
- **节省带宽**：为 mpv 增加 `--ytdl-format=bestaudio/best` 参数，禁止后台下载冗余的视频数据，彻底解决网络加载慢和卡顿问题。
- **数据库提速**：在 `schema.sql` 中为 `play_events` 和 `feedback_events` 表增加了 `occurred_at` 时间索引，防止随着听歌记录增加导致的查询卡顿。
- **稳健的 IPC 通信**：重构了 `_snapshot_mpv_slot`。增加了“重试机制”与“数据库降级查询(Fallback)”，即使网络流加载缓慢，也能确保打点标记和笔记功能不报错。
- **槽位管理修复**：修复了 `m stop` 失效的问题，确保所有 mpv 进程都能被准确注册到 `slots.json` 并被干净地回收。

### 🧠 智能搜索体系
- **多源并行搜索**：重构了搜索核心，采用 `ThreadPoolExecutor` 并发搜索 YouTube、Bilibili 和 SoundCloud。
- **智能源识别**：如果搜索词包含“哔哩哔哩/B站”，自动将 B 站权重置顶。
- **精准音乐匹配**：普通搜索自动追加 `" official audio"` 过滤掉杂乱的视频；如果搜“现场/演唱会”，则自动智能避让。

### 🎧 听感与功能升级
- **专业响度均衡 (Normalization)**：引入 `--af=loudnorm` 滤镜。无论来自哪个平台、年代多老的音源，播放时音量将始终保持一致，无需再手动调节。
- **无限电台模式 (`m radio`)**：基于当前播放曲目（或历史红心曲目）自动调用 YouTube 相关推荐接口，实现无限续播。
- **音乐心境日记 (`m note` & `m journal`)**：
  - 新增 `m note`：听歌有感时，一键记录感悟。自动抓取当前时间点（甚至能读取长视频的“章节名”如“蓝莲花”）。
  - 新增 `m journal`：按时间轴回顾所有伴随音乐产生的瞬间灵感。

### 📖 文档规范
- 新增 `COMMANDS.md` 命令行手册，梳理了所有控制、评价、推荐和系统维护指令。
